# selection-method.md — how stage 02 chooses what survives

Research casts wide; the piece is narrow. This is how you cut without distorting.

## The selection test

A finding survives if it does one of these, and is cut otherwise:
- **serves a deliverable** — directly helps deliver a D-item, or
- **carries the goal** — moves the reader toward what the goal wants, or
- **steelmans the other side** — is needed to state the strongest
  counterargument honestly.

A finding that is merely *interesting* does not survive. Interest is not a
deliverable; an unused fact is padding waiting to happen.

## Weighting by S-register

- A **load-bearing** claim — one a deliverable rests on — needs **primary or
  secondary** backing. If only a tertiary source supports it, either find
  better backing (flag back to stage 01) or down-rank the claim so the piece
  doesn't lean on it.
- A `contested` flag means the claim is presented *as* contested, with both
  sides' facts, or not presented as settled. Don't quietly pick a winner.
- A `dated` or `single-study` flag is carried forward so stage 03 can pitch the
  claim's confidence honestly.

## Building data tables

When several findings share a structure (same metric across cases, a trend over
time), assemble them into a table in the factsheet — columns, rows, and a source
S-tag per cell. This is what a stage-02 visual mark's `data=` points at. A table
in the factsheet is the precondition for a chart or table visual; no factsheet
table, no visual.

## Ordering for the outline

- Lead with the beat that stakes the goal, per the format (the op-ed's peg, the
  essay's stake, the explainer's question).
- Put the counterargument beat *before* the resolution it answers — a reader
  persuaded then ambushed feels manipulated.
- Order so the controlling metaphor can be introduced simply early and extended
  later (see imagery layering): don't front-load the hardest beat if it forces
  the metaphor to arrive fully grown.
- Distribute density evenly. The outline should make it *possible* to write the
  last third as richly as the first — if the back half is thin on beats, that's
  where stage 03 will be tempted to compress. Fix it here.
