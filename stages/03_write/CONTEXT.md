# Stage 03 — Write (execute the plan, cross the finish line)

*Contract. Runs after stage 02. This is where prose gets written — but only the
prose the outline licensed. It executes the argument, renders exactly the
imagery on the map, emits the visual specs the beats marked, and crosses the
**D-gate**: every deliverable present, length in band, no shortcuts. This is the
stage where lazy models fail, so its CHECK is a ritual, not a glance.*

**Read order:** your Layer 0 card → `../../CONTEXT.md` → this file. (LITE variant:
`CONTEXT_LITE.md`.)

---

## READ

- `../00_brief/output/brief.md` — goal, audience, format, length, `provocation`,
  the **D-list**.
- `../02_compile/output/factsheet.md` — the only facts you may claim, with their
  S-registers.
- `../02_compile/output/outline.md` — the beats, the `## Imagery` block, the
  per-beat visual marks. **This is your scope. Add nothing to it.**
- `_config/style/pinker-prose.md` — the prose floor.
- `_config/style/imagery.md` — how to render the licensed images.
- `_config/style/visuals.md` — **only if** visual marks exist; for the spec
  syntax.
- The active **audience** file — register and assumed knowledge.
- The active **format** file — structure, length, register defaults.
- `_config/rules.md` — the binding laws (respect floor, deliverables law,
  provenance, scope, voice).

## INTERNALIZE

You are casting the outline into prose at the prose floor, for this audience, in
this format, leaning as hard as the `provocation` dial allows and no harder. You
invent no facts (factsheet only), no images (imagery map only), no visuals (marks
only). The respect floor governs every line: write to the audience's ceiling.

## TRANSFORM — write the draft

- Render each outline beat in order, using only its F-tagged facts at or below
  their S-register.
- **Imagery:** introduce the controlling metaphor at its simplest early; extend
  (never replace) it as the argument deepens; render supporting analogies only
  in the declared domain. Nothing off the map.
- **Visuals:** at each marked beat, emit a typed spec in `visuals.md` syntax —
  `CHART:` / `DIAGRAM:` / `TABLE:` — with a `data:` reference to real F-tags and
  a one-sentence `message=`. Place it beside the claim it proves. Never write
  "insert graph here."
- **Forbidden phrases** (and the failure modes they signal): "in summary," "the
  remaining sections follow," "similarly," "and so on." The last third is
  written at the density of the first. No compressing later beats as the draft
  grows.

## WRITE

`output/draft.md` — the finished piece (or, for `format: course`, the cast
`course-package.md` per `course.md` §3).

## CHECK — the completion ritual (do not skip; reprint, don't assert)

Before STOP, reprint the **full D-list** and beside each item quote **one line of
evidence from your own draft** that satisfies it. No quote, no pass.

- [ ] D1 … D<n>: each present, each with a quoted line of evidence.
- [ ] Length inside the band (state the actual count).
- [ ] Every factual claim traces to an F-tag; no claim exceeds its S-register.
- [ ] Every image is on the imagery map; the controlling metaphor is single and
      consistent (not mixed, not abandoned).
- [ ] Every visual spec references real F-tags and sits beside its claim.
- [ ] None of the forbidden phrases appears; the last third is at full density.
- [ ] Respect floor held: nothing the audience's §3 says they know is
      re-explained; no condescension.

## STOP

Hand off to stage 04 (if `audit: on`) or to the human (if `audit: off`). If any
box is unticked — especially a D-item without a quoted line — you are not done.
Finish it. A model that won't finish has a finish line here it must physically
cross.
